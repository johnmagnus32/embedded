## Part 1: Two-level IRQ dispatch (Zephyr-style)

### How it works

1. The vector table has the **same function** (`_isr_wrapper`) at every IRQ slot
2. A **software ISR table** (generated by `gen_devicetree.py`) maps IRQ numbers to handler + arg
3. The dispatcher reads the active IRQ number from hardware, indexes the table, calls the handler

```
IRQ fires → ARM hardware jumps to _isr_wrapper
         → _isr_wrapper reads IRQ number from IPSR register
         → looks up _sw_isr_table[irq_num]
         → calls handler(arg)
```

### Why the indirection?

ARM vector table entries are bare function pointers — no way to pass an argument. But drivers need the device pointer as context. The dispatcher solves this by pairing each handler with its argument in the software table.

### Startup file (one-time rewrite, never changes again)

```asm
// rtos/arch/arm/startup_m4.s

.section .vector_table, "a"
.word _stack_top
.word reset_handler
.word 0                 /* NMI */
.word 0                 /* HardFault */
.word memmanage_handler
.word 0                 /* BusFault */
.word 0                 /* UsageFault */
.word 0, 0, 0, 0       /* Reserved */
.word svc_handler
.word 0, 0             /* Reserved */
.word pendsv_handler
.word systick_handler

/* All 82 peripheral IRQs point to the same dispatcher */
.rept 82
.word _isr_wrapper
.endr
```

That's it. No ISR names, no weak symbols, no per-peripheral entries. This file never needs editing regardless of which peripherals are used.

### Common dispatcher (new file: `rtos/arch/arm/irq_dispatch.c`)

```c
#include <stdint.h>
#include "irq.h"

/* Generated by gen_devicetree.py — defines _sw_isr_table[] */
#include "devicetree_irq.h"

void _isr_wrapper(void)
{
    /* Read active IRQ number from ARM IPSR register */
    uint32_t ipsr;
    __asm volatile("mrs %0, ipsr" : "=r"(ipsr));
    int irq = (int)ipsr - 16;  /* IPSR 16+ = IRQ 0+ */

    if (irq >= 0 && irq < NUM_IRQS && _sw_isr_table[irq].isr)
        _sw_isr_table[irq].isr(_sw_isr_table[irq].arg);
}
```

### IRQ header (`rtos/include/irq.h`)

```c
#ifndef IRQ_H
#define IRQ_H

#include <stdint.h>

#define NUM_IRQS 82

typedef void (*isr_fn)(void *arg);

struct isr_table_entry {
    isr_fn isr;
    void  *arg;
};

/* Generated table — defined in devicetree_irq.h */
extern const struct isr_table_entry _sw_isr_table[NUM_IRQS];

/* NVIC helpers */
static inline void irq_enable(uint8_t irq) {
    volatile uint32_t *iser = (volatile uint32_t *)(0xE000E100 + (irq / 32) * 4);
    *iser = (1 << (irq % 32));
}

static inline void irq_disable(uint8_t irq) {
    volatile uint32_t *icer = (volatile uint32_t *)(0xE000E180 + (irq / 32) * 4);
    *icer = (1 << (irq % 32));
}

#endif
```

### Generated ISR table (`gen_devicetree.py` output: `devicetree_irq.h`)

`gen_devicetree.py` scans the DTS for nodes with `irq = <N>` and `status = "okay"`, then generates:

```c
/* AUTO-GENERATED by gen_devicetree.py — DO NOT EDIT */
#ifndef DEVICETREE_IRQ_H
#define DEVICETREE_IRQ_H

#include "irq.h"

/* Forward declarations of driver ISRs */
extern void uart_stm32_isr(void *arg);
extern void spi_stm32_isr(void *arg);

/* Forward declarations of device structs */
extern const struct device __device_usart1;
extern const struct device __device_spi1;

/* Software ISR table */
const struct isr_table_entry _sw_isr_table[NUM_IRQS] = {
    [0 ... 81] = { .isr = (void *)0, .arg = (void *)0 },  /* default: unused */
    [37] = { .isr = uart_stm32_isr, .arg = (void *)&__device_usart1 },
    /* If SPI interrupt was needed:
    [35] = { .isr = spi_stm32_isr, .arg = (void *)&__device_spi1 },
    */
};

#endif
```

The mapping from `compatible` string to ISR function name follows a convention:
- `"st,stm32-usart"` → `uart_stm32_isr`
- `"st,stm32-spi"` → `spi_stm32_isr`
- `"st,stm32-i2s"` → `i2s_stm32_isr`

This convention is defined in `gen_devicetree.py` as a lookup table:

```python
ISR_NAMES = {
    "st,stm32-usart": "uart_stm32_isr",
    "st,stm32-spi":   "spi_stm32_isr",
    "st,stm32-i2s":   "i2s_stm32_isr",
    "st,stm32-dma":   "dma_stm32_isr",
}
```

### Driver changes (UART example)

```c
// uart_stm32.c — no hardcoded IRQ numbers, no ISR naming tricks

// The ISR — called by the dispatcher with the device pointer
void uart_stm32_isr(void *arg)
{
    const struct device *dev = (const struct device *)arg;
    const struct uart_stm32_config *cfg = dev->config;
    uint32_t sr = REG(cfg->base, USART_SR);
    if (sr & (1 << 5)) {
        char c = (char)REG(cfg->base, USART_DR);
        // ... buffer the byte ...
    }
}

// Init — just enables the IRQ, doesn't need to know the number at compile time
static int uart_stm32_init(const struct device *dev)
{
    const struct uart_stm32_config *cfg = dev->config;
    // ... pin config, baud rate setup ...

    // Enable the IRQ — number comes from DTS via generated config
    irq_enable(cfg->irq);
    return 0;
}
```

The `cfg->irq` field is populated from the generated devicetree header:

```c
static const struct uart_stm32_config uart_cfg_0 = {
    .base = DT_INST_ST_STM32_USART_0_REG_ADDR,
    .irq  = DT_INST_ST_STM32_USART_0_PROP_IRQ,  // 37 or 38, from DTS
    // ...
};
```

## Part 2: Flash driver uses DT_PARENT (no hardcoded SPI bus)

### Current (broken)

```c
// flash_w25q.c
DEVICE_DT_DECLARE(spi3);  // ← hardcoded!
static const struct device *spi(void) { return DEVICE_DT_GET(spi3); }
```

### New approach

The flash node is a child of its SPI bus in the device tree:

```dts
&spi3 {
    w25q128: flash@0 { compatible = "jedec,spi-nor"; ... };
};
```

`gen_devicetree.py` generates a parent reference:

```c
#define DT_INST_JEDEC_SPI_NOR_0_BUS_LABEL spi3
// or more generically:
#define DT_INST_JEDEC_SPI_NOR_0_PARENT_LABEL spi3
```

The flash driver uses it:

```c
// flash_w25q.c — no "spi1" or "spi3" anywhere
DEVICE_DT_DECLARE(DT_INST_JEDEC_SPI_NOR_0_PARENT_LABEL);

static const struct device *spi(void) {
    return DEVICE_DT_GET(DT_INST_JEDEC_SPI_NOR_0_PARENT_LABEL);
}
```

Move the flash node under `&spi1` in the DTS → `gen_devicetree.py` generates `_PARENT_LABEL spi1` → flash driver automatically uses SPI1. Zero code changes.
